## P0 — Fonctionnalité cassée

### cart/mini-cart — Hook `woocommerce_widget_shopping_cart_buttons` absent
- WC fire `do_action('woocommerce_widget_shopping_cart_buttons')` pour afficher les boutons "View cart" et "Checkout".
- Le thème hardcode un seul bouton "Checkout" sans fire ce hook.
- **Impact** : PayPal Express, Apple Pay, et tout plugin ajoutant des boutons via ce hook ne s'affichent pas.
- **Fix** : Ajouter `@php(do_action('woocommerce_widget_shopping_cart_buttons'))` ou fire le hook autour des boutons.

### cart/mini-cart — Null-safety `WC()->cart` manquant
- WC 10.0.0 vérifie `WC()->cart &&` avant `! WC()->cart->is_empty()`.
- Le thème omet cette vérification → fatal error possible si `WC()->cart` est null.

### cart/cart-shipping — Pas de `wc_cart_totals_shipping_method_label()`
- WC utilise `wc_cart_totals_shipping_method_label($method)` qui inclut le prix formaté avec taxes via le filtre `woocommerce_cart_shipping_method_full_label`.
- Le thème render manuellement `$method->get_label()` + prix custom → tout plugin hookant ce filtre est ignoré.

### cart/cart-shipping — Pas de hidden input pour méthode de livraison unique
- WC utilise `<input type="hidden">` quand une seule méthode est disponible.
- Le thème utilise toujours `<input type="radio">` → la méthode peut ne pas être soumise si l'utilisateur n'interagit pas.

### cart/cart-shipping — Paramètre manquant dans filtre
- `woocommerce_cart_no_shipping_available_html` : WC passe `($html, $formatted_destination)`, le thème passe seulement `($html)`.

### checkout/form-coupon — Pas de guard `wc_coupons_enabled()`
- Le formulaire coupon s'affiche même quand les coupons sont désactivés dans WC Settings.
- **Fix** : Ajouter `@if(wc_coupons_enabled())` autour du formulaire.

### checkout/form-coupon — Toggle et filtre manquants
- WC wraps le formulaire dans un mécanisme toggle avec `wc_print_notice()` et lien `showcoupon`.
- Le thème affiche le formulaire directement → filtre `woocommerce_checkout_coupon_message` contourné.
- Doublon `id="coupon_code"` dans le HTML (invalide).

### checkout/form-login — Formulaire toujours hidden après échec login
- WC 10.0.0 vérifie `$show_form = isset($_POST['login'])` pour garder le formulaire visible après erreur.
- Le thème hardcode `'hidden' => true` → après un échec de login, le formulaire se referme et l'utilisateur ne voit pas les erreurs.

### checkout/form-login — Logique de visibilité incomplète
- WC distingue `$registration_at_checkout` et `$login_reminder_at_checkout`.
- Le thème simplifie à un seul check `woocommerce_enable_checkout_login_reminder` → si registration=on mais login_reminder=off, WC afficherait le form mais le thème non.
- Filtre `woocommerce_checkout_login_message` absent.

### checkout/form-pay — Hook `woocommerce_pay_order_before_payment` absent
- WC 8.2.0 fire ce hook entre le tableau de commande et la section paiement.
- **Impact** : Plugins injectant du contenu avant le paiement ne fonctionnent pas.

### order/order-details — Boutons d'actions commande absents
- WC 10.1.0 génère des boutons d'action via `wc_get_account_orders_actions($order)` (payer, annuler, etc.).
- Le thème ne les affiche pas du tout.
- **Impact** : Le client ne peut pas agir sur ses commandes depuis la page détail.

### order/order-details-customer — Hook adresses absent
- `woocommerce_order_details_after_customer_address` (billing + shipping) absent.
- **Impact** : Plugins n° TVA, infos société ne s'affichent pas après les adresses.

### single-product/variable — Hook `woocommerce_after_variations_table` absent
- WC fire ce hook entre le tableau des variations et le `single_variation_wrap`.
- **Impact** : Plugins injectant du contenu après le tableau de variations cassés.

### global/quantity-input — Support `readonly` et `$type` absents
- WC 10.1.0 supporte `$readonly` et `$type` (text, number, etc.).
- Le thème hardcode `type="number"` et ignore `$readonly`.
- **Impact** : Certains types de produits nécessitant un input readonly ne fonctionnent pas correctement.

### single-product-reviews — Filtre `woocommerce_reviews_title` absent
- WC passe le titre à travers `apply_filters('woocommerce_reviews_title', $title, $count, $product)`.
- Le thème render directement le sprintf sans le filtre.

## P1 — Filtres / données contournés

### cart/cart — `product_name` non filtré dans quantity input
- WC passe `$product_name` (déjà filtré par `woocommerce_cart_item_name`) à `woocommerce_quantity_input()`.
- Le thème passe `$_product->get_name()` (non filtré) → le filtre `woocommerce_cart_item_name` est contourné pour l'attribut `product_name` du quantity input.

### cart/cart — Sold-individually bypass le template quantity
- WC utilise `woocommerce_quantity_input()` même pour les produits sold-individually (avec min=1, max=1).
- Le thème court-circuite avec un `sprintf('1 <input type="hidden" ...')` → tout plugin filtrant le quantity input pour les sold-individually est ignoré.

### cart/mini-cart — Filtre `woocommerce_widget_cart_item_quantity` reçoit mauvais type
- WC passe le markup `<span class="quantity">N × $price</span>`.
- Le thème passe un integer `$cart_item['quantity']`.

### cart/mini-cart — Attribut `data-success_message` absent sur lien remove
- WC 10.0.0 ajoute `data-success_message` pour les notices de suppression. Le JS WC peut en dépendre.

### checkout/thankyou — Sous-template `checkout/order-received.php` non appelé
- WC rend le message via `wc_get_template('checkout/order-received.php')`.
- Le thème inline le filtre `woocommerce_thankyou_order_received_text` directement.
- **Impact** : Tout plugin overridant `checkout/order-received.php` est contourné.

### checkout/payment — `is_ajax()` déprécié
- Déprécié depuis WC 6.1. Utiliser `wp_doing_ajax()`.

### myaccount/my-address — Hook `woocommerce_my_account_after_my_address` absent
- Ajouté en WC 8.7.0. Fire `do_action('woocommerce_my_account_after_my_address', $name)` dans le bloc adresse.
- **Impact** : Plugins ajoutant des champs après les adresses (TVA, etc.) ne fonctionnent pas.

### myaccount/form-edit-account — Hook `woocommerce_edit_account_form_fields` absent
- Ajouté en WC 8.7.0. Fire entre le champ email et le fieldset password.
- **Impact** : Plugins ajoutant des champs custom au formulaire de compte ne fonctionnent pas.

### myaccount/view-order — Filtre `woocommerce_order_details_status` absent
- Ajouté en WC 10.1.0. `apply_filters('woocommerce_order_details_status', ..., $order)`.

### myaccount/view-order — `wp_kses_post()` manquant sur notes
- WC utilise `wp_kses_post(wpautop(wptexturize($note->comment_content)))`.
- Le thème utilise `{!! wpautop(wptexturize($note->comment_content)) !!}` sans `wp_kses_post()`.

### order/tracking — Paramètre `$order` manquant dans filtre
- `apply_filters('woocommerce_order_tracking_status', sprintf(...))` → manque `$order` en 2e argument.

### order/order-details — `wc_wptexturize_order_note()` non utilisé
- Le thème utilise `wptexturize()` au lieu de `wc_wptexturize_order_note()`.

## P2 — Accessibilité

### cart/cart + cart/mini-cart
- `role="button"` absent sur les liens de suppression.
- `aria-label` statique "Remove this item" au lieu de "Remove %s from cart" (avec nom produit).

### cart/shipping-calculator
- `aria-expanded` et `aria-controls` absents sur le bouton toggle.
- `id="shipping-calculator-form"` absent sur le formulaire (requis par `aria-controls`).
- `<label>` absents pour country, state, city, postcode → non conforme WCAG.

### checkout/form-checkout
- `aria-label="Checkout"` absent sur le `<form>`.

### single-product/variable
- `role="presentation"` absent sur le tableau des variations.
- `aria-label` absent sur le lien "Clear" (reset variations).
- Div `screen-reader-text` avec `role="alert"` et `aria-live="polite"` absent (annonce reset aux lecteurs d'écran).

### single-product/grouped
- `<label>` absent sur les checkboxes sold-individually → pas de texte accessible pour les lecteurs d'écran.
- `id` absent sur l'input checkbox.

### single-product/product-attributes
- `aria-label="Product Details"` absent sur le tableau.
- `scope="row"` absent sur les `<th>`.

### global/quantity-input
- `aria-label="Product quantity"` absent sur l'input.
- `autocomplete` absent.

### global/form-login
- `required` et `aria-required="true"` absents sur les inputs.
- `aria-hidden="true"` absent sur `<span class="required">*</span>`.
- `<span class="screen-reader-text">Required</span>` absent.
- Variable `$hidden` ignorée (remplacée par Alpine.js modal).

### myaccount/navigation
- `aria-label="Account pages"` absent sur les `<nav>`.
- `aria-current="page"` absent sur le lien actif.

### myaccount/form-login
- `required` et `aria-required="true"` absents sur tous les champs requis.
- `<span class="screen-reader-text">Required</span>` absent.

### myaccount/form-lost-password
- `required` et `aria-required="true"` absents sur `user_login`.

### myaccount/form-reset-password
- `required` et `aria-required="true"` absents sur les deux champs password.

### myaccount/form-edit-account
- `aria-required="true"` absent sur les champs requis.
- `aria-describedby` absent sur `display_name`.
- `autocomplete` utilise `off` au lieu de `current-password` / `new-password`.

### myaccount/orders
- Colonne order-number devrait être `<th scope="row">` au lieu de `<td>`.
- `aria-label` absent sur les liens order number et les boutons d'action.

### loop/pagination
- `aria-label="Product Pagination"` absent sur `<nav>`.
- `end_size` et `mid_size` non passés à `paginate_links()` (WC utilise 3/3, le thème utilise les défauts WP 1/2).

### notices/notice
- `role="status"` absent (WC 10.4.0).

### single-product-reviews
- `role="heading"` et `aria-level="3"` absents sur le reply title.
- `autocomplete` absent sur les champs author/email.
